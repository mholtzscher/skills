# Phase 2 — Structures

## Goal

Design or change the core representations before APIs and behavior: data structures, schemas, state shapes, records, enums, storage layout, config shape, or protocol payloads.

## Allowed edits

- Structure/type/schema definitions
- Closely required imports or compile fixes for those definitions
- Workflow file updates

Do not implement behavior except what is necessary to keep structure-only edits coherent.

## Steps

1. Inspect existing structures and naming conventions.
2. Propose the smallest structure change that supports the goal.
3. Make phase-scoped edits.
4. Record rejected alternatives in the workflow file when useful.
5. Run project checks.
6. Run `plannotator review --git`.
7. Stop for explicit approval to enter phase 3.
